# OpenClaw_demo

开发团队需求主文档项目

## 项目结构

```
OpenClaw_demo/
├── agents/                 # Agent 配置文件
│   ├── 项目大肘子.md      # 需求分析师
│   ├── 编程大拿.md        # 开发工程师
│   └── 运维大师.md        # DevOps工程师
├── config/                 # 项目配置
│   └── team.json          # 团队配置
├── docs/                   # 文档目录
└── README.md
```

## 团队成员

| Agent | 角色 | 主要职责 |
|-------|------|----------|
| 项目大肘子 | 需求分析师 | 需求对接、需求设计、文档产出 |
| 编程大拿 | 开发工程师 | 技术设计、代码开发、测试回归 |
| 运维大师 | DevOps工程师 | CI/CD、部署、运维 |

## 工作流程

```
需求阶段 (项目大肘子)
    ↓
开发阶段 (编程大拿)
    ↓
部署阶段 (运维大师)
```

## 使用方式

### 调用子 Agent

可以通过 OpenClaw 的 sessions_spawn 来调用各个子 agent：

- **项目大肘子**: 负责需求分析和文档产出
- **编程大拿**: 负责代码开发和技术实现
- **运维大师**: 负责 CI/CD 和部署

### 配置修改

修改 `config/team.json` 来调整团队配置。
